# 12. セキュリティ（プロダクション必須）

## 1. 秘密情報 (Secrets)
- **漏洩防止**: APIキー・PW・トークンのハードコーディング/ログ出力禁止。
- **管理**: 環境変数や安全なストアを使用。`.env`はGit除外。機密情報は `[REDACTED]` 等でサニタイズ。

## 2. 入力・整合性 (Input & Integrity)
- **検証**: 全外部入力を検証（XSS/トラバーサル/長さ制限対策）。
- **SQLi**: 文字列結合禁止。パラメータ化クエリ/ORM必須。
- **認証・PW**: PWはソルト付ハッシュ化保存（bcrypt等）。JWT等は全リクエストで検証（期限/改ざん）。

## 3. インフラ・制限 (Infrastructure & Limits)
- **制限**: IP/ユーザー単位のレート制限。外部呼出のタイムアウト設定。
- **通信**: HTTPS必須、適切なヘッダー（HSTS, CSP等）設定。
- **Upload**: 拡張子/MIME/サイズ/ファイル名の厳格検証。

## 4. エラー・監視 (Logging & Monitoring)
- **ログ**: ユーザーへは汎用メッセージ、詳細は内部ログ。認証失敗・異常アクセス・権限エラーの記録。
- **依存性**: 脆弱性スキャン（npm audit / pip-audit等）の定期実施。

## 必須チェックリスト
- [ ] 秘密情報のハードコード・ログ漏洩なし
- [ ] 全外部入力の検証・サニタイズ
- [ ] パラメータ化クエリ/ORMの使用
- [ ] PWハッシュ化・JWT検証
- [ ] レート制限・タイムアウト設定
- [ ] HTTPS・セキュリティヘッダー設定
- [ ] 依存パッケージの脆弱性スキャン実施
