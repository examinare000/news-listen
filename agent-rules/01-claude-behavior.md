# 01. Claude Code固有の動作制約

Claude Code（claude.ai/code）に固有のルール。
全エージェント共通ルールは `03-agent-behavior.md`、Git は `10-git-strategy.md` を参照。

## 動作モード

Claude Code は以下の2モードで動作する。**作業開始時に必ずモードを判定**する。

| モード | 適用条件 | 参照ルール |
|---|---|---|
| **A. 複数LLM協調** | codex / gemini-cli MCP が有効 | `90-agentic-coding.md` |
| **B. Claude単体サブエージェント協調** | 上記が無効、または明示指示 | `91-claude-subagent-coding.md` |

モードBでは**メインClaudeはオーケストレーターに徹し、コーディング・レビュー・Git操作を直接行わない**。詳細は 91 を参照。

## タスク管理

- **TodoWriteの使用基準**:
  - 使用する: 3ステップ以上の複雑なタスク、複数ファイル横断、複数タスク列挙、テスト→実装→統合の段階的作業
  - 使用しない: 単一ファイルの簡単な修正、1〜2ステップで完了、情報収集のみ
- **同時in_progressは1件のみ**。完了したら即座にcompletedへ更新
- 実装中に発見したサブタスクはtodoとして追加

## レスポンス制約

- **簡潔性**: 4行以内を基本とする（コード・ツール出力は除く）
- 前置きや作業総括は省く。直接回答する
- ファイルパス・コマンド・識別子はバッククォートで明示

## ファイル操作

- 既存ファイル編集を新規作成より優先
- Edit/Write の前に必ず Read で内容確認
- 1 Edit = 1論理変更
- ドキュメントファイル（*.md）は明示的要求時のみ作成

## 検索・分析

- オープンエンドな検索は Task tool（subagent）を使用
- 複数の独立した情報収集は並列実行
- 推測でファイルを特定できる場合は直接 Read

## プロアクティブ実行ポリシー（モードA共通）

| プロアクティブに実行する | 実行しない |
|---|---|
| ファイル検索・分析 | 新規ファイル作成（特にドキュメント） |
| テスト実行 | git commit（明示的要求時のみ） |
| lint / typecheck | 設定ファイル変更 |
| セキュリティ検証 | 外部サービスアクセス |

## モードB時の追加制約

| 行為 | 直接実行 | 委譲先 |
|---|---|---|
| コーディング（Edit/Write） | ❌ | Coder |
| 実装プランの設計 | ❌ | Planner |
| コードレビュー | ❌ | Reviewer |
| Git操作 | ❌ | Git-composer |
| ユーザー応答 | ✅ メインのみ | - |
| タスク分解・指示 | ✅ メインのみ | - |

例外: 1〜2行の自明修正、Worker 完了後の整合性確認 Read は許容。

## 制約の継承

- `00-core-principles.md` の原則をすべて継承
- 矛盾時は番号が大きいファイルを優先
