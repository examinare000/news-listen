# 00-core-principles.md

## 絶対遵守の3原則
1. **デグレ防止**: 既存機能を破壊しない。追加・変更時は全既存テストの通過を確認。問題時は即座にロールバック。
2. **日本語統一**: 返答、コメント、エラー、テスト、コミット、ドキュメントは全て日本語。思考・検索のみ英語可（コードの命名は英語慣習に従う）。
3. **TDD（t-wada方式）**: Red ➔ Green ➔ Refactor サイクルを厳守。振る舞いに集中し、テストを仕様書とする（詳細: `11-testing-strategy.md`）。

## 開発サイクル（順序固定・変更禁止）
1. ブランチ作成（`10-git-strategy.md`）
2. ADR・設計更新（`30-documentation-management.md`）
3. テストファースト（`11-testing-strategy.md`）
4. TDDで実装
5. アトミックコミット（`10-git-strategy.md`）
6. 品質チェック（test / lint / typecheck）➔ マージ

## コミット前チェックリスト
- [ ] 全テスト通過
