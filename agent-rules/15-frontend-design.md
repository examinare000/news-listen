# 15. フロントエンドデザインガイドライン
「汎用的で個性のないデザイン」を避け、高品質で意図的なインターフェースを実装する。

## 設計思考プロセス
実装前に美的方向性を決定し、一貫して適用する。

### 検討事項
- **目的/ユーザー**: 解決する課題と対象者
- **美的トーン**: 方向性の選択
- **制約/差別化**: 技術要件（パフォーマンス、アクセシビリティ）と記憶に残る要素

### 美的方向性の選択肢
- **ブルータル・ミニマル**: 極限まで無駄を排除
- **マキシマリスト**: カオス的で豊富な情報と視覚要素
- **レトロ・フューチャリスティック**: 過去の未来観の導入
- **オーガニック/ナチュラル**: 有機的な曲線や自然形態
- **ラグジュアリー/リファインド**: 洗練された高級感
- **プレイフル/トイライク**: ポップで遊び心のあるデザイン
- **その他**: エディトリアル、幾何学的、ソフトパステル、インダストリアル

## 要素別ガイドライン

### タイポグラフィ
- Arial, Inter, Roboto, system-ui 等の**汎用フォントを避ける**。
- ディスプレイ用と本文用で、個性的かつ読みやすいフォントを組み合わせる。

### カラー＆テーマ
- CSS変数で一貫性を確保（`:root { --color-primary: #...; }`）。
- **支配的な色と鮮明なアクセント**の配色を推奨。
- 「紫グラデーション on 白背景」などの陳腐な配色を避ける。

### モーション・アニメーション
- **高インパクトな瞬間**（ページロード時のスタッガードリビール等）にアニメーションを集中。
- CSSのみの実装を優先。React等ではMotionライブラリを活用。
- スクロールトリガーやホバー状態を効果的に使用。

### 空間構成＆ディテール
- **レイアウト**: 非対称性、オーバーラップ、対角線、グリッド破り、意図的な余白/密度の管理。
- **背景/装飾**: 単色背景を避け、グラデーションメッシュ、ノイズテクスチャ、幾何学パターン、半透明、ドラマチックなシャドウ、装飾ボーダー等で深みを出す。

## 禁止事項
- **フォント**: Inter, Roboto, Arial, システムフォント, Space Grotesk
- **配色**: 紫グラデーション on 白背景
- **レイアウト**: 予測可能な汎用テンプレート
- **全般**: 個性のないクッキーカッターデザイン
