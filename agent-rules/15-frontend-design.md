# 15. フロントエンドデザインガイドライン

高品質で記憶に残るフロントエンドインターフェースを作成するためのガイドラインです。
AIによる「汎用的で個性のないデザイン」を避け、プロダクション品質のコードを実装します。

## 設計思考プロセス

コーディング前に、文脈を理解し**明確な美的方向性**を決定します。

### 検討事項

| 観点 | 質問 |
|------|------|
| 目的 | このインターフェースが解決する問題は何か？誰が使うか？ |
| トーン | どの美的方向性を選ぶか？（下記参照） |
| 制約 | 技術要件（フレームワーク、パフォーマンス、アクセシビリティ） |
| 差別化 | 何がこのデザインを**記憶に残る**ものにするか？ |

### 美的方向性の選択肢

- **ブルータル・ミニマル**: 極限まで削ぎ落とした無駄のないデザイン
- **マキシマリスト**: 情報と視覚要素が豊富なカオス的デザイン
- **レトロ・フューチャリスティック**: 過去の未来観を取り入れたデザイン
- **オーガニック/ナチュラル**: 自然の形や有機的な曲線
- **ラグジュアリー/リファインド**: 洗練された高級感
- **プレイフル/トイライク**: 遊び心のあるポップなデザイン
- **エディトリアル/マガジン**: 雑誌のような編集的レイアウト
- **アールデコ/ジオメトリック**: 幾何学的パターン
- **ソフト/パステル**: 柔らかい色調
- **インダストリアル**: 工業的・機能的デザイン

**重要**: 明確なコンセプト方向を選び、一貫して実行します。大胆なマキシマリズムも洗練されたミニマリズムも有効です。鍵は**意図的であること**です。

## 実装品質基準

実装するコード（HTML/CSS/JS、React、Vue等）は以下を満たすこと:

- プロダクション品質で機能的
- 視覚的に印象的で記憶に残る
- 一貫した美的視点を持つ
- すべての細部に配慮

## デザイン要素別ガイドライン

### タイポグラフィ

```css
/* ❌ 避けるべき汎用フォント */
font-family: Arial, sans-serif;
font-family: Inter, sans-serif;
font-family: Roboto, sans-serif;
font-family: system-ui, sans-serif;

/* ✅ 個性的で記憶に残るフォント選択 */
/* ディスプレイフォントとボディフォントを組み合わせる */
```

- **美しく、ユニークで、興味深いフォント**を選択
- 汎用フォント（Arial、Inter、Roboto）を避ける
- 個性的なディスプレイフォントと洗練されたボディフォントを組み合わせる

### カラー＆テーマ

```css
/* ✅ 推奨: CSS変数で一貫性を保つ */
:root {
  --color-primary: #...;
  --color-accent: #...;
  --color-background: #...;
}
```

- 一貫した美的を追求
- CSS変数で統一性を確保
- **支配的な色と鮮明なアクセント**が、控えめで均等に分散した配色より効果的
- 紫のグラデーション on 白背景などの陳腐な配色を避ける

### モーション・アニメーション

```css
/* ✅ 高インパクトな瞬間に集中 */
/* ページロード時のスタッガードリビール */
.element {
  animation: fadeIn 0.5s ease-out;
  animation-delay: var(--stagger-delay);
}
```

- エフェクトとマイクロインタラクションにアニメーションを使用
- HTMLではCSS-onlyソリューションを優先
- Reactでは利用可能な場合Motionライブラリを使用
- **高インパクトな瞬間に集中**: スタッガードリビール付きの一つのページロードが、散在したマイクロインタラクションより効果的
- スクロールトリガーや驚きを与えるホバー状態を活用

### 空間構成

- 予想外のレイアウト
- 非対称性
- オーバーラップ
- 対角線的なフロー
- グリッドを破る要素
- 余白を活かすOR密度をコントロール

### 背景と視覚的ディテール

単色背景をデフォルトにせず、雰囲気と深みを作成:

- グラデーションメッシュ
- ノイズテクスチャ
- 幾何学パターン
- レイヤー化された透明度
- ドラマチックなシャドウ
- 装飾的なボーダー
- カスタムカーソル
- グレインオーバーレイ

## 禁止事項

以下の「AIっぽい」汎用的な美学を**絶対に使用しない**:

| カテゴリ | 避けるべき例 |
|----------|--------------|
| フォント | Inter、Roboto、Arial、システムフォント、Space Grotesk |
| 配色 | 紫グラデーション on 白背景 |
| レイアウト | 予測可能なレイアウトとコンポーネントパターン |
| 全般 | コンテキスト固有の個性がないクッキーカッターデザイン |

## 実装の複雑さとビジョンのマッチング

- **マキシマリストデザイン** → 豊富なアニメーションとエフェクトを含む精巧なコード
- **ミニマリスト/洗練デザイン** → 抑制、精密さ、スペーシング・タイポグラフィ・微細なディテールへの注意

エレガンスは、ビジョンを適切に実行することから生まれます。

## 創造性の原則

各デザインは異なるべきです:
- ライトテーマとダークテーマを変化させる
- 異なるフォントを使用
- 異なる美学を採用
- 世代を超えて共通の選択に収束**しない**

**AIエージェントは並外れた創造的な仕事ができます。枠にとらわれず考え、独自のビジョンに完全にコミットしたとき、真に何が創造できるかを示してください。**
