# 03. エージェント共通動作ルール

Claude / OpenAI / Gemini など任意のAIエージェントが本リポジトリで一貫した品質で作業するための共通ルール。

## プラン・進捗管理

- 着手前に短いプラン（3〜7手順）を提示する
- 1〜2文のプレアンブルで「これから何をするか」を述べる
- 進行中は1件のみ。完了したら即座に更新

## 応答スタイル

- 簡潔・手順志向・事実ベース。不要な説明は省く
- ファイルパス・コマンド・コード識別子はバッククォートで明示
- 出力は「意図 → 操作 → 結果 → 次の手」の順で要約

## ファイル操作

- 読み → 編集 → 検証 の順。差分は最小に保つ
- 新規ファイルは目的と配置理由を1文で記す
- 既存仕様を壊す変更は避け、拡張で実装できるか検討

## 実行・検証

- ローカルなテスト・lint・型チェックから狭く検証し、徐々に広げる
- 破壊的操作（削除・上書き・リセット）は理由・影響・ロールバック案を事前提示
- 実行不能な環境では検証コマンドと期待結果を明記

## セキュリティ

- 認証情報・個人情報・機密の新規作成や露出は禁止
- 必要時はダミーや `.env.example` を用意
- 外部通信・インストールは目的・最小権限・代替案を提示してから
- 詳細は `12-security-guidelines.md`

## デバッグ・ログ

- 一時的なデバッグ出力は最小限。完了時に専用コミットで削除
- プレフィックス `[DEBUG]` `[TRACE]` を使用
- コメントアウトではなく完全削除

## モデル別補足

- **Claude Code**: 思考と出力を分離し、出力は結果に限定。差分は小さく、危険操作は事前宣言。詳細は `01-claude-behavior.md`
- **OpenAI（GPT/o系）**: ツール連携がない場合はパッチ適用手順を明記。長大ファイルは分割提示
- **Gemini系**: 書き込み制限がある場合は提案 → 承認 → 適用の順。生成物の保存先を明示

**適用優先度**: 🟠 高（全エージェントで遵守）
