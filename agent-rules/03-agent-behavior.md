# 03. エージェント共通動作ルール

## プラン・進捗管理
- 着手前に短プラン（3〜7手順）および「これから何をするか」を1〜2文で提示。
- 進行中は1件、完了後即更新。

## 応答スタイル
- 簡潔・事実ベース。不要説明は省略。
- パス、コマンド、識別子はバッククォート表記。
- 出力は「意図 ➔ 操作 ➔ 結果 ➔ 次の手」で要約。

## ファイル操作
- 読込 ➔ 編集 ➔ 検証を遵守。差分は最小限。
- 新規ファイル作成時は目的と理由を1文で記載。
- 既存破壊を避け、拡張実装を検討。

## 実行・検証
- テスト/lint/型チェックで段階的検証。
- 破壊操作（削除/上書き）は理由、影響、リカバリ案を事前提示。
- 実行不能環境では検証コマンドと期待結果を明記。

## セキュリティ
- 機密情報（認証・個人情報等）の作成/露出禁止。`.env.example`等を使用。
- 外部通信/モジュール導入は目的、最小権限、代替案提示後。詳細: `12-security-guidelines.md`

## デバッグ・ログ
- 一時出力は最小限。完了時にコミットから完全削除（コメントアウト不可）。
- `[DEBUG]` `[TRACE]` 接頭辞を使用。

## モデル別補足
- **Claude**: 思考/出力分離、差分最小、危険操作は事前宣言。詳細: `01-claude-behavior.md`
- **OpenAI**: ツール無時はパッチ手順明記、大ファイルは分割提示。
- **Gemini**: 書込制限時は提案 ➔ 承認 ➔ 適用。保存先明示。
